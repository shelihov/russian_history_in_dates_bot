                    Постановка задачи

    Что?
Телеграм бот, в котором можно заучить даты из Истории России, путем решения тестов.

    Чтобы что?
Чтобы обучающиеся могли в интерактивной и быстрой для усвоения форме выучить даты, к примеру, для подготовки к ЕГЭ.

    Что бот должен уметь?
1. По запросу отправлять пользователю клавиатуру с выбором направления подготовки.
2. Отправлять случайный вопрос из выбранного направления и прикрепленную к нему клавиатуру: 1 правильный варинт, 3 ложных.
   Фиксировать статистику, вопросы не повторяются
3. После решения одного направления, отправлять статистику и нерешенные вопросы

    Дополнительный функционал
Не предусмотрен

    Описание взаимодействия с ботом
1. Пользователь отправляет команду /start боту (или стартует его, найдя в поиске)
2. Бот приветствует пользователя и предлагает выбрать направление, отправляя клавиатуру с вариантами
   направления подготовки: "ВРЕМЯ ПРАВЛЕНИЯ", "БИТВЫ", "ВОЙНЫ И ВОССТАНИЯ", "РЕФОРМЫ" а также предлагает
   пользователю прочитать подробные правила, отправив команду /help
3. На этом этапе пользователь может совершить 3 действия:

    1. Согласится решать тесты, выбрав один из 4х тестов
    2. Отправить в чат команду /help
    3. Отправить в чат любое другое сообщение

4. Пользователь выбирает одно из направлений подготовки на клавиатуре:

    1. Используя логику бота и модуль random, бот отправляет в чат один из вопросов и клавиатуру к нему.
       Пользователь делает выбор, бот возвращает ключ из словаря ответов, сверяет их и затем:

        1. Если пользователь отвечает правильно:

            1. Бот отправляет сообщение в чат: 'Ответ верный'.
            2. Бот в группе состояний ставит +1 правильный ответ.
            3. Добавляет ключ и значение в словарь в группе состояний
            4. Отправляет клавиатуру с одной кнопкой - "Решать дальше?" - "Да"

        2. Если пользователь отвечает неправильно:

            1. Бот отправляет сообщение в чат: 'Ошибка! Ответ неверный.'.
            2. Бот в группе состояний ставит +1 неправильный ответ.
            3. Добавляет ключ и значение в словарь в группе состояний
            4. Отправляет клавиатуру с одной кнопкой - "Решать дальше?" - "Да"
            5. Добавляет ключ в виде текста в список неверных ответов в группу состояний, с помощью .append

    2. Затем бот сравнивает извлекаемый ключ из подготовленных вопросов с ключами в словаре группы состояний,
       и если они не равны х != y, то отправляет новый вопрос в чат, операция продолжается до тех пор, пока
       х не будет равен y.
    3. После перебора всех элементов в словаре, бот отправляет пользователю сообщение, в котором указывает
       сколько всего вопросов, колличество верных ответов, ошибок, а так же список с ошибочными вопросами,
       каждый с новой строки.
    4. Отправляет клавиатуру - 'Вернутся к выбору направления?' - 'Да', 'Нет'.

5. Пользователь отправляет в чат команду /help:

    1. Бот присылает в чат информацию и описание функционала бота, а так же клавиатуру 'Вернутся к выбору направления?' - 'Да', 'Нет'"

6. Пользователь отправляет в чат любое другое сообщение:

    1. Бот присылает в чат сообщение, что не понимает пользователя.
